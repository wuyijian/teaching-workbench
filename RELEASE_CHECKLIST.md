# 发版验证清单（Release Checklist）

每次打包 / 发版前，**必须**按此清单验证一遍。第一阶段是脚本化的快速 smoke，第二阶段是端到端 UI 验证。

---

## 阶段一：自动化 Smoke（必跑）

```powershell
npm run smoke
```

`scripts/release-smoke.mjs` 会顺序跑：

| # | 检查项 | 通过标准 |
|---|---|---|
| 1 | `tsc --noEmit` | TypeScript 零错误 |
| 2 | 讯飞 `office-api-ist-dx.iflyaisol.com/v2/upload` 鉴权探测 | HTTP 200，返回 `code:000000` 或 `110001(fileSize too small)` |
| 3 | 大模型 `/chat/completions` 非流式 | 返回 `choices[0].message.content` |
| 4 | 大模型 `/chat/completions` 流式 | 至少收到 1 个 chunk |
| 5 | Supabase auth 健康检查（可选） | `auth/v1/health` 返回 2xx |

任何一项不通过，**立即停止发版**，先修。

也可以单独跑：
- `npm run smoke:xfyun` — 只验讯飞
- `npm run smoke:llm` — 只验大模型

## 阶段二：核心功能端到端验证（人工）

启动方式（任选）：
```powershell
npm run dev              # 浏览器版
npm run electron:dev     # 桌面端（推荐每次发版都跑一遍）
```

### A. 转写能力 ⭐ 核心

**A1 — 短音频（< 1 分钟）冒烟**

- [ ] 上传一段 30s 左右的中文录音（mp3/m4a/wav）
- [ ] 任务卡片显示「上传中 → 转写中 → 已完成」
- [ ] 进度百分比正常前进
- [ ] 转写结果有正确的时间戳分段
- [ ] 多上传 2 个任务，验证「队列机制」（只有一个在跑，其余排队）

**A2 — 长音频（> 30 分钟）压力**

- [ ] 上传一段 30~60 分钟的录音
- [ ] 不报「ts illegal / signa fail」
- [ ] 自适应轮询间隔生效（不疯狂打日志）
- [ ] 最终能拿到完整结果

**A3 — 异常场景**

- [ ] 取消按钮：进行中的任务能被取消，状态变为「已取消」
- [ ] 重试按钮：失败任务能重新走一遍流程
- [ ] 删除任务：清单 + localStorage 同步移除
- [ ] 关闭页面后再打开：进行中的任务被标为 `error: 页面刷新后转写中断`，已完成的保留

**A4 — 配额扣量（仅 Web 版有限额）**

- [ ] 转写完成后右上角配额条数字下降
- [ ] 配额用尽后，新建任务被拒绝并弹「升级方案」

---

### B. 反馈生成能力 ⭐ 核心

**B1 — 基础生成**

- [ ] 选中一个已完成任务，点「生成反馈」
- [ ] Markdown 实时流式渲染
- [ ] 完成后能「复制」「保存」

**B2 — 生成不被打断（关键回归）**

- [ ] 点「生成反馈」开始流式 → 切到另一个任务 → 切回 → 内容仍在累加，下拉里显示「生成中…」徽标
- [ ] 切到「内容问答」tab → 切回 → 仍在生成
- [ ] 切到「学生档案」/「AI 助手」顶层模式 → 切回 → 仍在生成
- [ ] 多任务并发：同时给 A、B 两个任务点生成 → 互不影响

**B3 — 自定义提示词**

- [ ] 设置 → 反馈提示词改一个明显能识别的话术 → 重新生成 → 输出风格变化

**B4 — 补充信息**

- [ ] 在 FeedbackPanel 顶部展开「补充信息」，写「课前测平均 78 分」
- [ ] 重新生成 → 反馈中能体现这个补充信息
- [ ] 切走再回来，补充信息还在（自动保存）

**B5 — 追问**

- [ ] 已有 feedback 的情况下输入追问 → 流式回复
- [ ] 追问中切任务 → 同样不被打断

**B6 — 异常**

- [ ] 关掉网络再点「生成反馈」→ 显示明确的网络错误
- [ ] 「停止」按钮可以中止生成

## 出现问题怎么办

| 故障类型 | 排查文档 |
|---|---|
| 讯飞 26601 / signa fail | 检查 `.env` 的 `VITE_XF_*` 三件套，跑 `npm run smoke:xfyun` |
| LLM 401 / 403 | 检查 `VITE_LLM_API_KEY`、`VITE_LLM_BASE_URL` 是否完整域名 |
| 反馈生成切任务后中断 | 检查 `FeedbackPanel.tsx` 的 `byTask` 状态 + `RightPanel.tsx`/`App.tsx` 的 `hidden` 切换是否还在 |
| Supabase 提交建议失败 | 在 Supabase SQL Editor 执行 `supabase/migrations/005_feedback_bucket.sql` |
| 兑换激活码失败 | 在 Supabase SQL Editor 执行 003、002、004 三个迁移 |
